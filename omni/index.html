# Omni App — Day View Handover: Time-Driven Timetable ## What Needs to Change (and What Stays the Same) **Keep everything about the current Omni Day view** — Free rows, current/next highlighting, the dot colors, the + button for personal events, Today's Habits section below. All of that stays. **What needs fixing:** 1. Work events with `customStartTime`/`customEndTime` show wrong times — use the custom times 2. Span events (e.g. 15:10–15:30 crossing two slots) show as two rows — merge into one 3. Personal events from `omni_personal_slots` should slot into the timeline by actual time --- ## Data Sources ### 1. Work Events — Supabase `work_app_data` table Read two keys: **`slotTimes`** — period boundaries (stored as JSON string): ```json [{"t":"07:00"},{"t":"07:30"},{"t":"08:00"},{"t":"08:20"},{"t":"09:05"}, {"t":"09:50"},{"t":"10:35"},{"t":"11:00"},{"t":"11:45"},{"t":"12:30"}, {"t":"13:30"},{"t":"14:15"},{"t":"15:00"},{"t":"15:15"},{"t":"15:30"}, {"t":"16:15"},{"t":"17:00"},{"t":"18:00"},{"t":"18:45"},{"t":"19:15"}, {"t":"20:30"},{"t":"21:30"},{"t":"22:30"},{"t":"23:00"}] ``` **`ttCustom`** — all custom events (stored as JSON string), keyed `"weekOffset_slotIdx_day"`: ```json { "5_19_Fri": { "l": "Freddd", "tp": "go", "cat": "go", "recur": "once", "placedWeek": 5, "customStartTime": "15:10", "customEndTime": "15:30", "endSlotIdx": null, "isSubSlot": false } } ``` **Day names:** `Mon`, `Tue`, `Wed`, `Thu`, `Fri`, `Sat`, `Sun` --- ## Computing Real Start/End Times For each `ttCustom` entry matching today's day and week: ```js const [wOffsetStr, slotIdxStr, day] = key.split('_'); const slotIdx = parseInt(slotIdxStr); const event = ttCustom[key]; // Always prefer custom times over slot defaults const realStart = event.customStartTime || slotTimes[slotIdx]?.t; const realEnd = event.customEndTime || slotTimes[slotIdx + 1]?.t || null; ``` If `customEndTime` is set, that IS the real end time — it may extend beyond the next slot boundary. --- ## Recurrence Matching ```js function eventMatchesWeek(event, currentWeekOffset) { if (event.recur === 'once') return event.placedWeek === currentWeekOffset; if (event.recur === 'weekly') return true; if (event.recur === 'biweekly') return (currentWeekOffset - event.placedWeek) % 2 === 0; return false; } ``` --- ## Span Merging — Skip Continuation Slots When an event has `customEndTime` that goes past the next slot boundary, the raw `ttCustom` only has ONE entry (the starting slot). The continuation slots are just empty in `ttCustom`. So when building the event list, skip any slot whose time falls within a previous event's range: ```js const skippedSlots = new Set(); const dayEvents = []; for (let i = 0; i < slotTimes.length; i++) { if (skippedSlots.has(i)) continue; const key = `${currentWeekOffset}_${i}_${currentDayName}`; const event = ttCustom[key] || null; const base = /* your base slot data for this slot/day */; const cell = event || base; if (!cell) continue; const realStart = event?.customStartTime || slotTimes[i]?.t; const realEnd = event?.customEndTime || slotTimes[i + 1]?.t; // Mark continuation slots to skip if (event?.customEndTime) { const endM = t2m(event.customEndTime); for (let j = i + 1; j < slotTimes.length; j++) { if (t2m(slotTimes[j].t) < endM) skippedSlots.add(j); else break; } } dayEvents.push({ slotIdx: i, cell, realStart, realEnd, isWork: !!event }); } ``` --- ## 2. Personal Events — `omni_personal_slots` table Columns: `id`, `title`, `start_time` (HH:MM), `end_time` (HH:MM), `recurrence` (`once`/`weekly`/`daily`), `date` (YYYY-MM-DD), `day_of_week` (Mon/Tue/etc) ```js function personalMatchesDay(p, dateStr, dayName) { if (p.recurrence === 'daily') return true; if (p.recurrence === 'weekly') return p.day_of_week === dayName; if (p.recurrence === 'once') return p.date === dateStr; return false; } ``` Personal events have a full `title` and are shown with their own color in Omni. --- ## Unified Timeline Merge work events and personal events, sort by start time: ```js const allEvents = [ ...dayEvents.map(e => ({ type: 'work', start: e.realStart, end: e.realEnd, label: e.cell.l || '', cat: e.cell.cat || e.cell.tp || 'free', slotIdx: e.slotIdx })), ...personalSlots .filter(p => personalMatchesDay(p, todayDateStr, currentDayName)) .map(p => ({ type: 'personal', start: p.start_time, end: p.end_time, label: p.title, cat: 'personal' })) ]; allEvents.sort((a, b) => a.start.localeCompare(b.start)); ``` Render each event exactly as Omni currently renders timetable rows — same dot colors, same Free row style, same current/next highlighting — just with real HH:MM times. --- ## Category Colors (Work events) | cat | color | |---|---| | `lesson` | `#1d4ed8` blue | | `duty` | `#7c3aed` purple | | `go` | `#d97706` orange | | `tmtg` | `#0d9488` teal | | `support` | `#0e7490` cyan | | `empty`/`free`/`sleep` | grey (current Free style) | Legacy aliases: `free`/`sleep`/`personal` → `empty`, `mtg` → `go`, `abs`/`zpl` → `support` --- ## Key Principles - **Keep Free rows** — show them exactly as they appear now in Omni - **No P1/P2/P3 labels** — only HH:MM times - **customStartTime/customEndTime are the source of truth** — always prefer over slot defaults - **Personal events are first-class** — same timeline, sorted by time, full title shown - **Span events show as one row** — skip continuation slots, use the real end time --- ## Supabase Read ```js const { data } = await supabase .from('work_app_data') .select('key, value') .in('key', ['ttCustom', 'slotTimes']);
